让APP数据统计如此简单

发表于 2016-04-16   |     |   阅读次数

现在APP开发都会加数据统计的功能,比如

  • 点登录btn的事件发生的次数
  • 在好友界面停留多少时间
  • 修改头像从选择到上传成功用了多长时间

统计的数量和项目,业务需求不一样个数也就不一样 从几十到几百不等.
现在做统计平台的有不少,比如友盟,诸葛ios ,FLURRY，GA等,就要在加入这些SDK 的时候要注意一点广告ID追踪码的问题 要不有可能会被苹果拒了

下面先以友盟按上面的三个需求看一下代码
友盟iOS 统计集成文档
下面先按上面的三个需求对应的代码

1,点登录btn的事件发生的次数

2, 在好友界面停留多少时间

3,个改头像从选择到上传成功用了多长时间

花几个小时 加了50个onclick 事件,30个界面的停留时间长,20个自定义事件.完成 抽个烟准备下班.
等等,产品经理过来了,小王有几个新统计文档已经发你企邮了如下

  • 新加的那个会员功能也要在加20个左右事件.

  • 所有统计都同步给我们自己的server发一份好和第三方广告平台和统计平台做对比

  • 所有的统计在给诸葛io发一份 他们会根据用户的操作形为做有价值分析

其实刚才我们加的那些统计代码是有一个问题的

1,代码在各自的控制器,类里,VIEW 等和业务逻辑都参杂在一起,到处都是统计的代码,耦合度太高不好维护 当改版或者重构时，有可能忘了把相应的打点代码迁移过去(我原来就出现过多次 三个月前的统计然后改版本了,对应的统计没有加上,数据报表就会有问题)

2,对于如上需求就不太好处理了

所以我对统计这块做了一些整理 相关的代码demo 已经上传github
主要思路是用 AOP(Aspect-Oriented-Programming)，就是「面向切面编程」
就是可以动态的在函数调用的前后插一段代码。iOS 可以使用 Pete Steinberger 开发的 Aspects 这个库，大致原理是在 runtime 层，通过 swizzle method 来实现的

工具篇之pdf批量转xcassets

发表于 2016-04-16   |     |   阅读次数

instruction

写这个工具的目的是为了更高效率使用PDF资源,降低手动操作的错误率.主要功能 批量把 pdf 转成 xcassest
现在有的小伙伴可能已经支持 目前做的好一点的做是这样设计和工程代码在不同的分支.
如果新加一个图片资源做法大约是这样:

  • 设计师提交到svn分支
  • 研发人员更新资源
  • 研发人员拖到xcassets中

用这个工具设计师在出PDF资源后 直接生成研究人员人的xcassets 提交到分支,开发人员更新就可以得到最新资源,这个工具主要是给设计师用的.
目录结构说明:

./bin 编译好的可执行文件

./pdfConvertXcassets 主要代码

./sample 例子pdf 文件

How to use

设置输入 和输出path 点开始就可以了
